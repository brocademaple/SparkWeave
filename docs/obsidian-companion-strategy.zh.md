# SparkWeave 作为 Obsidian Companion

## 一句话定位

SparkWeave 应该成为 Obsidian 用户的伴侣层：Obsidian 继续承担长期、本地、可迁移的知识库；SparkWeave 承担个性化灵感编织 Agent，负责解释关系、形成判断，并把零散笔记推向可创作的产物。

这个产品承诺可以概括为：

```text
Obsidian 保存用户的思考资产。
SparkWeave 帮用户看见这些资产正在指向什么。
```

## 生存位：不做第二个 Obsidian

SparkWeave 不应该进入 Obsidian 最强的领域：

- Markdown 编辑
- 双链和图谱导航
- Canvas 式自由排布
- 通用笔记数据库
- 插件市场
- 同步、发布或知识库托管
- 要求用户迁移整个知识库

它的生存位应该更窄也更锋利：面向已经拥有笔记、摘录、碎片和未完成想法的创作者，提供解释、孵化和行动建议。

## 为什么这个方向仍然成立

Obsidian 已经具备很强的基础能力：

- 本地优先的 Markdown vault
- 双链和未链接提及
- 图谱视图
- Canvas
- 标签、属性和 Bases
- 搜索
- 社区插件
- TypeScript 插件 API
- Obsidian URI 自动化

这些能力帮助用户保存和导航知识。SparkWeave 的价值在于更主动、更有主张：

- 发现反复出现的主题。
- 解释灵感为什么连接。
- 区分证据、冲突、因果、延展和行动。
- 把一个主题簇转成下一步 brief。
- 学习用户真正认可的解释和输出形式。

## 可以参考 Obsidian 的能力

### 链接是核心资产

链接应该被当成长期创作资产。SparkWeave 需要保留原始笔记引用，让生成结果可以追溯，并尽量以可迁移的 Markdown 写回。

可以参考 Obsidian：

- 双链
- 出链
- 可用时保留块引用或标题引用
- 标签和属性

SparkWeave 的延展：

- 把链接转化为关系解释。
- 说明某个链接为什么影响当前创作目标。
- 展示某个链接如何改变判断或下一步行动。

### 未链接提及

未链接提及很重要，因为用户不可能记得所有潜在连接。SparkWeave 可以用 Agent 感知扩展这个能力：

- 发现重复出现的人名、短语、主题和未解决问题。
- 在用户行动之前准备前置内容。
- 推荐哪些材料值得在写作、规划或决策前先看一眼。

关键变化是从“这里也出现过这个词”，推进到“这条旧材料可能改变你接下来该做什么”。

### Canvas 与关系标签

Obsidian Canvas 说明很多用户会用空间关系思考。SparkWeave 可以借鉴连接线标签，但要把标签变成产品逻辑的一部分。

当前内置关系主类：

- 因果
- 证据
- 冲突
- 延展
- 行动

未来可考虑的关系主类：

- 对照
- 先后
- 约束
- 例子
- 问题
- 反例
- 来源
- 待验证假设

### 本地优先

本地优先值得保留，因为用户的笔记是数字资产。用户应该可以在离开 SparkWeave 时带走自己沉淀出的价值。

最低期待：

- 导出 Markdown
- 导出 JSON
- 保留来源链接
- 保留关系标签
- 保留主题簇总结
- 保留反馈历史
- 保留生成产物

## SparkWeave 的差异化

SparkWeave 应该在一个具体方向上比 Obsidian 更主动：帮助用户把知识网络收束成判断和行动。

差异化产品面：

- 从零散笔记中发现主题。
- 解释碎片之间的关系。
- 每个 Weave Cluster 都给出 Action Advice。
- 提供按创作者类型定制的输出模板。
- 根据反馈持续个性化。
- 连接 Obsidian、Notion、语雀、印象笔记等现有笔记工具。

这会让 SparkWeave 更接近创作孵化 Agent，而非新的笔记软件。

## 边类型治理

关系标签很有价值，但完全开放自定义可能导致标签失控。如果用户创造太多只适用于单个案例的标签，长期图谱会变得难以阅读和检索。

推荐策略：

1. 保留一组少量的内置关系主类。
2. 允许用户自定义标签，但每个自定义标签都要映射到一个主类。
3. 当某个标签过窄或只使用一次时，给出提醒。
4. 对相似标签建议合并。
5. 展示用户的原始自定义标签，但搜索、筛选和 Agent 推理优先使用主类。

示例：

```text
用户自定义标签：数字员工落地卡权限
映射主类：约束
```

这样既给高级用户灵活度，也避免长期图谱不可用。

## Action Advice 与跟进验收

Action Advice 只有在产品能帮助用户轻量跟进时才真正有价值，但这种跟进不能变成强迫。

MVP 阶段的跟进应该轻：

- 每个主题簇只给一个建议动作。
- 用户可以确认、保存、延期或拒绝。
- 输出后允许评分和文字反馈。
- 记录建议动作是否转成了素材、产物或项目。

未来可扩展：

- 今天 / 本周节奏。
- Agent 生成验收标准。
- 小型 checklist。
- “这个行动之后发生了什么？”复盘。
- 可选 Daily Assistant 提醒。

默认应该由用户轻确认。Agent 可以辅助判断和复盘，但不应该强行接管用户的节奏。

## Local-first 与资产迁移

SparkWeave 应该默认承认：用户未来可能会把数据迁移到别处。这是健康的产品设计。

可导出的资产类型：

- 捕捉素材
- 素材类型树
- 原始笔记引用
- 关系边
- Weave Cluster
- 建议动作
- 反馈记录
- 生成产物
- 创作者偏好

优先导出格式：

- Markdown：给用户阅读和归档。
- JSON：用于结构化迁移。
- CSV：用于简单复盘和分析。

## Connector Roadmap

### Phase 1：Obsidian

- 导入选中的 Markdown 笔记。
- 解析标签、链接、属性和标题。
- 生成主题 brief。
- 把生成 Markdown 写回 SparkWeave 文件夹。
- 需要时通过 Obsidian URI 打开生成笔记。

### Phase 2：Notion、语雀、印象笔记

- 导入页面或选中文档。
- 保留来源 URL 和标题。
- 统一转成 `material_fragment`。
- 第一版避免做完整双向同步产品。

### Phase 3：Agent Review

- 对选中来源做定时或手动复盘。
- 发现值得处理的主题簇。
- 在写作或决策前推荐前置内容。
- 尊重 Daily Assistant 和主动建议开关。

### Phase 4：插件、MCP 或本地桥接

- Obsidian 插件命令：Weave selected notes。
- MCP 或本地桥接，用于受控地搜索 vault 和写入笔记。
- 把 connector 行为暴露成明确工具。

## MVP 案例：为什么收藏越多，产出越少？

这个选题可以作为第一个 Obsidian companion MVP 案例。

输入：

- 5-10 条关于收藏资料、未完成想法、模糊灵感、任务管理和产出焦虑的笔记。
- 来源可以是 Obsidian 导入，也可以先手动粘贴。

SparkWeave 输出：

- 主题：收藏带来的虚假进步感。
- 连接原因：这些笔记共同描述了“捕捉 -> 堆积 -> 缺乏组织 -> 没有行动”的链条。
- 共同问题：想法被保存了，但没有被组织成结构。
- 建议动作：写一篇短文《为什么收藏越多，产出越少？》。

文章结构：

1. 收藏带来的虚假进步感。
2. 真正的问题是想法没有被组织。
3. 好工具应该帮人发现主题，而不只是保存资料。

## 推荐 MVP 架构

先从导入/导出和简单 Obsidian handoff 开始。

```text
Obsidian vault 或手动粘贴笔记
  -> SparkWeave 导入选中素材
  -> SparkWeave 生成主题簇和解释
  -> 用户接受、编辑或拒绝 brief
  -> SparkWeave 导出 Markdown / JSON
  -> 用户可把结果保存回 Obsidian
```

这样可以先验证价值，再决定是否投入完整 Obsidian 插件。

## 产品 Presentation 文案

短 pitch：

```text
SparkWeave 把零散创作笔记转化为可解释的主题、判断和下一步行动。
```

痛点：

```text
创作者收藏越多，产出越少，因为保存下来的想法很少会自动组织自己。
```

差异化：

```text
Obsidian 帮你保留知识网络。SparkWeave 帮你判断这个网络今天意味着什么。
```

## 产品原则

1. Obsidian 是第一优先来源。
2. SparkWeave 是解释和行动层。
3. 不要求用户迁移。
4. 每个生成连接都必须有原因。
5. 每个 Weave Cluster 都必须有建议动作。
6. 用可迁移格式写回。
7. AI 输出是可编辑草稿，不是真理。
8. 用户反馈会影响未来检索、Prompt 和模板。

## 关键假设

- 第一批用户已经有笔记、摘录、碎片或草稿。
- 最强切入点是主题发现和行动转化。
- Obsidian 是第一优先 connector。
- Notion、语雀、印象笔记先作为导入路线记录。
- 关系标签应该支持有治理的自定义。
- Action Advice 第一版用用户轻确认、评分和文字反馈来验收。
